# API 契约说明：宪法原则 20/21 合规改造

## 说明

本需求为**内部实现合规性改造**，不新增、不修改任何对外 HTTP 接口或 Feign 契约。

- **不新增**：无新端点、无新请求/响应体。
- **不修改**：既有接口的 URL、方法、请求参数、响应体结构、状态码、错误码均保持不变。
- **改造范围**：仅 Service 层内部的对象转换方式（原则 20）与参数/空值处理方式（原则 21）。

因此本目录下**无需**维护 OpenAPI/Feign 等契约文件。若需验证行为，请直接对既有接口做回归（参见 [quickstart.md](../quickstart.md)）。
