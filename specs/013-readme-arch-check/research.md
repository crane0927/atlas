# 调研记录

## 决策

### 核对基准

- **Decision**: 以项目宪法中模块职责边界与架构规范作为核对基准
- **Rationale**: 宪法为当前项目的统一规范来源，可作为一致性判断依据
- **Alternatives considered**: 仅以历史 README 互相对照（缺少规范基准，结论不稳定）

### 判断规则

- **Decision**: 采用“技术栈版本 + 模块职责边界 + 依赖约束”三类规则判定一致性
- **Rationale**: 三类规则覆盖最易失真与最关键的架构约束，且便于追踪差异
- **Rules**:
  - 技术栈版本必须与宪法一致（JDK 21、Spring Boot 3.5.9、Spring Cloud 2025.0.1、Spring Cloud Alibaba 2025.0.0.0）
  - 模块职责描述需与现有模块划分一致（网关/认证/系统服务/公共模块/API 契约）
  - API 模块仅保留接口契约与 DTO，不包含实现依赖或实现层描述
  - 公共基础设施模块不应承载业务逻辑描述或跨模块职责

### 输出形式

- **Decision**: 以文档清单记录核对结果与差异点
- **Rationale**: 便于追踪与复核，支持后续修订
- **Alternatives considered**: 直接修改 README（不在本次范围内）
