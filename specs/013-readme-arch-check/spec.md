## 功能描述

对项目内各模块 README.md 进行一致性核对，判断其内容是否符合当前项目架构与模块职责边界，并形成可追踪的核对结果，便于后续修订与治理。

### 用户场景
- 架构维护者需要确认各模块 README 是否与当前架构规范一致
- 模块负责人需要获取 README 不一致项清单以便修订

### 功能需求
1. 覆盖所有模块 README.md，逐一核对与当前项目架构描述一致性
2. 记录每个 README 的一致/不一致结论与差异点
3. 输出统一的核对结果清单，便于追踪与复核

### 范围
- 包含：README 内容与架构/职责边界/模块归属的核对
- 不包含：自动修改 README 内容、变更代码或接口

### 依赖与假设
- 项目当前架构规范已在文档中明确且可作为核对基准
- 各模块 README.md 可访问且存在

### 成功标准
- 100% 模块 README 完成核对并记录结果
- 不一致项具备可追踪描述，支持后续修订

## 业务逻辑

- 汇总待核对的 README 清单
- 逐个核对内容与架构规范一致性
- 输出核对结果与差异记录

## 异常处理

- README 缺失：记录缺失项并继续其他核对
- 架构基准不明确：记录为待确认项
