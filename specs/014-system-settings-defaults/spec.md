## 功能描述

为 atlas-system 增加“系统默认设置”管理能力，保留系统内置默认设置，提供按 key-value 形式维护配置。设置项按类型分为“系统类型”和“自定义类型”：系统类型允许修改 value，但不允许删除；自定义类型允许修改与删除。该功能面向系统管理员，用于统一维护系统配置项，避免误删关键默认设置。

### 用户场景与验收

1. 管理员查看默认设置列表，可区分系统类型与自定义类型。
2. 管理员编辑系统类型的 value 成功，其他字段不可变更。
3. 管理员新增自定义类型设置项成功，可在列表中查看。
4. 管理员删除自定义类型设置项成功；删除系统类型时被拒绝并提示原因。

### 功能需求

1. 系统默认设置清单始终可见，且系统类型不可删除。
2. 设置项以 key-value 形式呈现，支持按类型筛选与快速定位。
3. 系统类型设置项仅允许修改 value，不允许删除与修改 key。
4. 自定义类型设置项允许新增、修改 value 与删除。
5. key 需唯一，新增时需校验重复并给出清晰提示。
6. 所有变更需记录操作者与时间以便追溯。

### 成功标准

- 90% 的管理员在 2 分钟内完成“新增自定义设置项”的任务。
- 删除系统类型设置项的操作 100% 被拦截并给出明确原因。
- 默认设置列表加载成功率达到 99% 以上。

### 假设与约束

- 只有系统管理员可进行新增、删除和修改操作；其他用户只读。
- 默认设置项在系统内置，且不会被本功能移除。

## API 设计

### 接口列表

| 方法 | 路径 | 描述 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| 查询 | 系统设置/默认配置 | 查看设置项列表（含类型） | 查询条件（类型、关键字） | 设置项列表 |
| 新增 | 系统设置/默认配置 | 新增自定义类型设置项 | key、value | 新增结果 |
| 修改 | 系统设置/默认配置 | 修改设置项 value | key、value | 修改结果 |
| 删除 | 系统设置/默认配置 | 删除自定义类型设置项 | key | 删除结果 |

### 请求/响应示例

示例（概念）：

- 查询返回：[{ key: "site.name", value: "Atlas", type: "系统类型" }, { key: "theme.color", value: "blue", type: "自定义类型" }]
- 新增请求：{ key: "theme.color", value: "blue", type: "自定义类型" }
- 修改请求：{ key: "site.name", value: "Atlas Pro" }
- 删除请求：{ key: "theme.color" }

## 数据模型

### 关键实体

- 设置项（数据库实体需继承 `BaseEntity`，审计字段来自基础实体）
  - key：配置项唯一标识
  - value：配置值
  - type：系统类型 / 自定义类型
  - createdBy / updatedBy：操作者（来自 `BaseEntity`）
  - createdAt / updatedAt：操作时间（来自 `BaseEntity`）

### 关系

- 一个设置项对应一个唯一 key；系统类型与自定义类型互斥。

## 业务逻辑

1. 列表展示时同时呈现系统类型与自定义类型，并标识不可删除状态。
2. 修改操作只允许更新 value；若为系统类型，禁止删除与修改 key。
3. 删除操作仅对自定义类型生效；系统类型删除请求直接拒绝。
4. 新增自定义类型时校验 key 唯一性，重复则拒绝并提示原因。
5. 所有变更行为记录操作者与时间，确保可审计。

## 异常处理

- 尝试删除系统类型设置项：拒绝并提示“系统默认设置不可删除”。
- key 重复：拒绝并提示“key 已存在，请更换后再试”。
- 无权限操作：拒绝并提示“无操作权限”。
- value 为空或不合法：拒绝并提示“value 不符合要求”。
