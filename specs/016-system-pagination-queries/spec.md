# 功能规格说明：通用分页 DTO 与 atlas-system 分页查询接口

## 宪法检查

本规格说明必须符合项目宪法要求：

- ✅ **RESTful API**: 所有接口遵循 RESTful 设计规范，统一使用 `Result<T>` 包装
- ✅ **中文注释**: 所有代码使用中文注释
- ✅ **代码复用**: 通用分页 DTO 供多模块复用
- ✅ **模块化**: 功能归属正确模块（common 与 atlas-system）
- ✅ **实体继承**: 数据库实体必须继承 BaseEntity
- ✅ **DTO/VO 规范**: 分页请求/响应对象遵循项目 DTO、VO 与 PageResult 规范

## 功能概述

### 目标

1. 提供一套**通用的分页查询请求规范**，使所有需要分页的接口在请求参数和语义上保持一致。
2. 为 **atlas-system** 中的主要业务表（用户、角色、权限、系统设置）提供**分页查询接口**，支持前端列表页按页加载、排序与条件筛选。

### 业务价值

- **一致体验**：所有列表页使用统一的分页参数（页码、每页条数、排序），降低前端与调用方理解成本。
- **可维护性**：分页逻辑与数据结构统一后，后续新增分页接口或调整规则时改动集中、影响可控。
- **可扩展性**：通用分页请求可复用到其他微服务或模块，避免各模块各自定义分页参数。

## API 设计概要

本需求涉及以下能力，具体路径与请求/响应结构需符合项目 RESTful 与 Result/PageResult 规范：

| 能力 | 方法 | 描述 | 响应 |
|------|------|------|------|
| 用户分页列表 | GET | 分页查询用户列表，支持条件与排序 | 分页结果（列表 + 分页元数据） |
| 角色分页列表 | GET | 分页查询角色列表，支持条件与排序 | 分页结果（列表 + 分页元数据） |
| 权限分页列表 | GET | 分页查询权限列表，支持条件与排序 | 分页结果（列表 + 分页元数据） |
| 系统设置分页列表 | GET | 分页查询系统设置列表，与通用规范一致 | 分页结果（列表 + 分页元数据） |

请求参数（各接口共用）：当前页、每页大小、可选排序字段与方向、各业务可选筛选条件。

## 涉及范围

### 通用分页能力（可复用）

- 定义统一的**分页查询请求**规范（如：当前页、每页大小、排序字段与方向）。
- 该规范可供 atlas-system 及未来其他模块共同使用，避免重复定义。

### atlas-system 分页查询接口

| 业务域     | 对应数据表/实体 | 说明                         |
|------------|------------------|------------------------------|
| 用户       | 用户表           | 用户列表分页查询             |
| 角色       | 角色表           | 角色列表分页查询             |
| 权限       | 权限表           | 权限列表分页查询             |
| 系统设置   | 系统设置表       | 系统设置列表分页查询（已有则对齐通用规范） |

说明：用户-角色、角色-权限等关联表以“关联管理”接口为主，本需求不单独为关联表提供分页列表接口。

## 功能需求

### FR-1: 通用分页请求规范

- **FR-1.1**：存在统一的“分页请求”定义，包含：当前页码、每页条数。
- **FR-1.2**：支持可选的排序信息（排序字段、升序/降序），以便不同列表复用同一套排序语义。
- **FR-1.3**：分页请求可与各业务查询条件组合使用（如按用户名、状态筛选用户），不替代业务条件，只统一分页与排序部分。

### FR-2: 用户列表分页查询

- **FR-2.1**：提供用户列表的分页查询接口，支持按页码、每页条数返回一页用户数据。
- **FR-2.2**：支持可选的查询条件（如用户名模糊、状态等），具体字段以业务需要为准。
- **FR-2.3**：响应中除列表外，包含总记录数、当前页、每页大小等分页信息，便于前端分页组件展示。
- **FR-2.4**：支持按指定字段排序（如创建时间、用户名等）。

### FR-3: 角色列表分页查询

- **FR-3.1**：提供角色列表的分页查询接口，支持按页码、每页条数返回一页角色数据。
- **FR-3.2**：支持可选的查询条件（如角色名称、角色编码等）。
- **FR-3.3**：响应包含分页信息及可选的排序支持。

### FR-4: 权限列表分页查询

- **FR-4.1**：提供权限列表的分页查询接口，支持按页码、每页条数返回一页权限数据。
- **FR-4.2**：支持可选的查询条件（如权限编码、权限名称等）。
- **FR-4.3**：响应包含分页信息及可选的排序支持。

### FR-5: 系统设置列表分页查询

- **FR-5.1**：系统设置已具备分页查询能力时，其请求参数与响应格式应与通用分页请求规范及统一分页响应保持一致。
- **FR-5.2**：若当前实现与通用规范不一致，需在本需求中统一为通用分页请求与统一分页响应。

## 用户场景与验收

### 场景 1：管理端查看用户列表（分页）

- **角色**：管理员
- **目标**：在用户管理页按页查看用户列表，并可按条件筛选、排序。
- **流程**：打开用户管理 → 选择页码或每页条数 → 可选输入用户名/状态等条件 → 请求分页接口 → 展示当前页用户及分页信息。
- **验收**：接口返回当前页用户数据及总条数、当前页、每页大小；条件与排序生效。

### 场景 2：管理端查看角色列表（分页）

- **角色**：管理员
- **目标**：在角色管理页按页查看角色列表，支持条件与排序。
- **流程**：打开角色管理 → 分页/条件/排序 → 请求角色分页接口 → 展示当前页角色及分页信息。
- **验收**：接口返回当前页角色数据及完整分页信息；条件与排序生效。

### 场景 3：管理端查看权限列表（分页）

- **角色**：管理员
- **目标**：在权限管理页按页查看权限列表，支持条件与排序。
- **流程**：打开权限管理 → 分页/条件/排序 → 请求权限分页接口 → 展示当前页权限及分页信息。
- **验收**：接口返回当前页权限数据及完整分页信息；条件与排序生效。

### 场景 4：管理端查看系统设置列表（分页）

- **角色**：管理员
- **目标**：在系统设置页按页查看设置项，与其它列表使用一致的分页与响应格式。
- **流程**：打开系统设置 → 分页/条件 → 请求系统设置分页接口 → 展示当前页及分页信息。
- **验收**：接口使用通用分页请求参数及统一分页响应；行为与用户/角色/权限分页一致。

## 成功标准

| 编号 | 描述 | 衡量方式 |
|------|------|----------|
| SC-1 | 存在可复用的通用分页请求定义 | 至少一个模块（如 common 或 system）提供该定义，且被多个分页接口使用 |
| SC-2 | 用户、角色、权限、系统设置均提供分页查询接口 | 四个业务域各有至少一个分页列表接口 |
| SC-3 | 所有分页接口的请求参数语义一致 | 页码、每页条数、排序等含义统一，文档或实现可验证 |
| SC-4 | 所有分页接口的响应包含列表与分页元数据 | 至少包含：当前页数据、总记录数、当前页、每页大小（或总页数） |
| SC-5 | 列表页可在 3 秒内完成单次分页请求（常规数据量） | 在常规数据量下（如单表万级以内）接口响应时间满足要求 |

## 假设与约束

### 假设

- 前端或调用方通过 HTTP 查询参数或请求体传递分页参数（页码、每页大小、排序等）。
- 已有统一的分页响应结构（如包含 list、total、page、size 等），本需求只要求请求侧统一，并确保响应侧与现有规范一致。
- 用户、角色、权限、系统设置等列表需做权限控制，具体鉴权方式由现有网关/认证体系决定，本需求不改变现有鉴权策略。

### 约束

- 分页请求与响应需符合项目宪法中的 RESTful、DTO/VO 及 Result/PageResult 规范。
- 不改变现有非分页接口的 URL 或契约；仅新增或调整分页相关接口与 DTO。

## 范围边界

### 包含

- 通用分页请求 DTO（或等价规范）的定义与使用方式。
- atlas-system 中用户、角色、权限、系统设置四类实体的分页查询接口。
- 上述接口的请求参数（分页 + 可选业务条件）、响应格式（列表 + 分页元数据）及排序行为。
- 与现有 SystemSetting 分页实现的对齐（若存在）。

### 不包含

- 用户-角色、角色-权限等关联表的分页列表接口（关联管理保持现有设计）。
- 其他微服务（非 atlas-system）的分页接口实现，仅约定通用 DTO 可被复用。
- 导出、批量操作、复杂报表等与“分页列表”无直接关系的功能。
